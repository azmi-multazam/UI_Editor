ReadMe


- Biasakan backup sebelum merubah SystemUI

- Aplikasi ini udah otomatis membuat folder UI_Editor/fonts di SdCard

- kalo abis restart, photo di statusbar agak lama munculnya karena system harus menscan dulu cache aplikasi.
beda ama app nemesis yang tidak menggunakan cache karena hanya mengirim perintah aja.
Kalo App UiEditor ini pake cache dulu lalu dikirim ke statusbar.

- Photo layout FB di statusbar akan hilang sementara ketika agan selesai pilih photo galery dan akan muncul setelah agan tap "Set Photo". ini berlaku di photo profile dan cover di tab PROFIL tapi tidak untuk tab OTHER.

- "Set Photo" menyetel photo menjadi photo profile di layout FBui tapi tidak menyimpan photo, dan "Save Photo" menyimpan photo di sdCard dan otomatis akan merubah Layout FB ui statusbar.

- tidak ada yang sempurna, apalagi hasil nyubi gan.. kalo ada bug, silahkan informasikan.

Installasi


- Buat yg udah make UI_Editor.apk v.1, tinggal masukin smali.v2 lalu ganti id publiknya ama id publik yang ada di smali.v1, jadi catet dulu id publicnya sebelum dihapus.
- Sebetulnya di v.2 ini cuma mengupdate smali text aja, jadi smali buat gambar/photo yg di folder cover, owner dan statusbar tetep pake v.1 jg ngga apa2.
LINK smali.v2 ada dibawah.
=============

- Buat pengguna baru silahkan ikuti langkahnya dibawah ini :

Asumsi agan udah pake FB ui + aplikasi Lenox & Nemesis:

1. Decompile SystemUI.apk

2. Buka layout FBui agan, perhatikan line dibawah ini:

<com.lenox.Lenox_Label.lenoxpicture android:id="@id/lenoxpic" ...../>

<com.nemesis.ownerinfo.Carrier android:id="@id/pp" ...../>

<com.nemesis.ownerinfo.NamePlate android:id="@id/profile_name" .... />

<com.lenox.Lenox_Label.lenoxlabel android:id="@id/lenoxlabel" .... />

Ganti adress smalinya jadi begini :

<com.zam.cover.Cover android:id="@id/lenoxpic" ... />

<com.zam.owner.ProfilePicture android:id="@id/pp" ... />

<com.zam.owner.ProfileName android:id="@id/profile_name" ... />

<com.zam.cover.Quote android="@id/lenoxlabel" ... />

NOTE : android="@id/..." ga usah diganti supaya ga usah recompile dan decompile lagi. Abaikan @id/.. kalo beda ama punya agan.


4. Buka public.xml, cari id yang tertera di code atas tadi, id ini akan di pake ama smali nantinya :

<public type="id" name="lenoxpic" id="0x7f0XXXXX" />
<public type="id" name="profile_name" id="0x7f0XXXXX" />
<public type="id" name="lenoxlabel" id="0x7f0XXXXX" />
<public type="id" name="pp" id="0x7f0XXXXX" />


5. Download smalinya di sini

6. Extract, masukin 3 folder ke folder smalinya SystemUI

7. Penggantian id smali
- Buka smali/com/zam/cover/Cover.smali, cari :
const v4, 0x7f0c00e2
ganti code id-nya ama id lenoxpic punya agan

- Buka smali/com/zam/cover/Quote.smali, cari :
const v6, 0x7f0c00e4
ganti code id-nya ama id lenoxlabel punya agan

- Buka smali/com/zam/owner/ProfileName.smali, cari :
const v6, 0x7f0c00e3
ganti code id-nya ama id profile_name punya agan

- Buka smali/com/zam/owner/ProfilePicture.smali, cari :
const v4, 0x7f0c00e5
ganti code id-nya ama id pp punya agan

8. Kalo untuk layout FBui selesai ampe sini, recompile, push ke system.
Tapi kalo pengen nambah lainnya, lanjut ke step berikut :

9. Smali yang ada di folder /com/zam/statusbar/... bisa di pake dimana aja, misalnya dipake buat ngisi tab kosong di expanded statusbar.

tinggal atur adress smalinya seperti ini

penempatan gambar
<com.zam.statusbar.Carrier1 android:id="@+id/carrier1" ... />

<com.zam.statusbar.Carrier2 android:id="@+id/carrier2" ... />

<com.zam.statusbar.Carrier3 android:id="@id/carrier3" ... />


penempatan text
<com.zam.statusbar.Text1 android:id="@+id/text1" ... />

<com.zam.statusbar.Text2 android:id="@+id/text2" ... />

<com.zam.statusbar.Text3 android:id="@+id/text3" ... />


Recompile untuk mendapatkan id public baru, decompile lagi, lalu buka smali/com/zam/statusbar/Carrier1.smali, ganti id publicnya ama id punya agan.

Contoh:
<public type="id" name="carrier1" id="0x7f0c003e" />

maka buka Carrier1.smali, cari :
const v4, 0x7f0c003e dan ganti id-nya ama punya agan. Dan begitu seterusnya.

10. TAMAT

Issues


= Di sebagian device yg pake Gingerbread aplikasi ini ngga berjalan sempurna karena statusbar tidak menampilkan photo profile dan cover. Ini 'mungkin' disebabkan karena SystemUI GB ga ada izin buat akses ke sdcard, begitu pula dengan sebagian device ICS dengan masalah yg sama.

- solusinya, buka AndroidManifest.xml, masukan line ini

<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" />
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />


taro di line ke 4 atau di jajaran <uses-permission...
ini dicoba lo yaaa... saya jg ga yakin dengan solusi ini karena ngga punya GB :p atau bisa juga coba2 compare AndroidManifest.xml punya GB ama punya JB.

= JIKA aplikasi FC ataw loading lama ketika proses cropping cover, coba ubah settingan cover quality ke medium atau low.

= JIKA size, color dan style ga work mungkin smalinya belum diganti ama smali.v2,
kl udah diganti tp tetep ngga work, teliti lagi,
kl udah yakin bener tp tetep ga work kemungkinan ngga support. atau compare xmlnya ama yg work
